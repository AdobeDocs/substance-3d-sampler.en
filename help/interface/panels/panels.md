# Panels

>[!IMPORTANT]
>
> Future change
> 
> Support for environment lights and meshes will be removed with the release of Sampler V5.2

Each panel in Sampler is dedicated to a specific purpose. By default, only the following four panels are open:

* <b>Project panel</b>: Allows you to manage assets in the currently opened project.
* <b>Assets panel</b>: Holds starter content and other asset libraries that you can use in the creation of your assets.
* <b>Layers panel</b>: Displays the layer stack of the selected asset.
* <b>Properties panel</b>: Shows properties and parameters for the currently selected layer - the <b>Layers </b>and <b>Properties panels</b> are used together

These panels are most regularly used in the process of creating assets.

The remaining panels are closed by default, and hold options and controls that generally don't require regular changes. You can access these panels from the left and right sidebars.

<table>
<tr style="border: 0;">
<td style="border: 0;" valign="top">

The following panels are stored at the bottom of the <b>left sidebar</b> by default:

* <b>Generative (BETA) panel</b>: Use text prompts to create textures or patterns with AI.
* <b>Shader Settings panel</b>: Allows you to modify shader parameters that can help improve performance or appearance of some materials.
* <b>Channel Settings panel</b>: Manage channels for your material.

</td>
<td style="border: 0;" valign="top">

The following panels are stored at the top of the <b>right sidebar</b> by default:

* <b>Exposed Parameters panel</b>: Allow you to manage the parameters that will be available when your asset is exported as SBSAR or SBS files.
* <b>Physical Size panel</b>: Configure the real-life physical size of your scanned samples and images.
* <b>Metadata panel</b>: Edit metadata by adding a description or changing the author name for the selected asset.
* <b>Export panel</b>: Send your asset directly to other applications or export it for general use.

</td>
</tr>
</table>

You can customize the workspace by moving panels - [see the interface page to find out how](../../interface/interface.md).

To return to the default workspace layout at any point use **Window &gt; Reset to default layout**.
